# UX / UI Audit — 2026-07-19

Independent pass over every interactive surface, requested by the user
("audit all buttons, settings, layouts, shortcuts, menus, aesthetics — then
overhaul"). Method: full read of the UI layer (`+page.svelte`, every
component, `app.css`), a shortcut-conflict map, a context-menu coverage map,
and a progress-feedback review. Fixes landed in the same session unless
marked *deferred*.

## What was already in good shape (no churn)

- **Theme system**: four coherent themes on CSS custom properties; neutral
  graphite default is the right call for judging photos. Consistent use of
  `color-mix` for states. Left alone.
- **Context menu component**: custom, clamped to viewport, closes on every
  reasonable signal; native webview menu globally suppressed except in text
  inputs — no "Save image / Reload" leakage anywhere.
- **Media context menu** (grid / filmstrip / Focus): thorough — nav, stacks,
  view jumps, open/reveal, pick/reject/clear, exports, copy path, with
  multi-selection counts. Folder tree has its own menu (open, refresh, paste
  moved files, reveal, copy path, subfolder scope).
- **Toolbar structure**: logical grouping (View · Arrange · Filters · Zoom ·
  actions right), labels above groups, popovers instead of control sprawl.
  The 2026-07 audits' earlier bar rework holds up.
- **Keyboard model**: no true conflicts found. `X/P/U/1–5/`/6–9` culling,
  `G/D/F/L/I` views, `Enter` focus toggle, `,/./[/]/Space` video, Ctrl-chords
  for files/undo; Escape cascade well-ordered (fullscreen → dim → view →
  selection). Shift+arrows correctly reserved for selection everywhere.

## Defects found → fixed

| # | Finding | Fix |
|---|---|---|
| 1 | **Toolbar popovers (Settings, Filters, Arrange, Clear, Cast) had no light dismiss** — they only closed by re-clicking their toggle; clicking elsewhere or pressing Esc left them hanging. The single biggest "feels un-native" wart. | Global pointerdown light-dismiss + Esc closes the open popover first (before any other Esc behavior). Details-view Columns menu got the same treatment. |
| 2 | **Details view rows had no context menu** (grid/filmstrip/Focus did). | Rows now open the same media menu (`onrowcontext`). |
| 3 | **No shortcut discoverability** — ~30 shortcuts existed, documented only in scattered tooltips and one settings hint line. | New `?` overlay: every key, grouped (Navigate / Views / Culling / Video / Files / mouse & controller pointers). Entry in Settings too. |
| 4 | **No ETA anywhere** — activity chips showed counts/percent only; the Prepare button's estimate was a naive linear extrapolation that lies badly on photo+video folders (videos cost ~15× per item). | Activity store now computes sliding-window rate ETAs (shown only after ~2.5 s of stable data; never on indeterminate jobs). Prepare processes **photos first, then videos**, with per-phase measured rates + sane priors, so its ETA is honest from the start and the chip shows `n / m · ~4m 30s`. |
| 5 | **Keyboard focus was invisible** — no `:focus-visible` styling anywhere; tabbing gave no visual feedback. | Global focus ring (`--select`, only on `:focus-visible` so mouse clicks stay clean). |
| 6 | Details "Marks" column rendered stars as ASCII `*` (everywhere else uses ★). | ★. |
| 7 | Context-menu glyph risks: `⿻` (CJK ideograph — tofu risk on lean font stacks) for RAW→JPEG; plain letter `E` for Open in Edit. | `⤓` and `✎`. |
| 8 | **Empty states were dead ends**: filtered-to-zero said "Nothing here matches" with no way out; the welcome screen was bare text. | Filtered-empty now shows the folder count + a **Clear filters** button (new one-click reset of all filter criteria). Welcome shows the fox mark + starter key hints. |
| 9 | Tooltips didn't teach keys. | View chips, Reject, info-bar Pick/Reject/stars now carry their shortcut in the tooltip. Prepare's tooltip rewritten for what it actually does now (photos *and* video posters + scrub strips, phased, with ETA). |
| 10 | Edit-mode Look panel: preset relevance and slider sensitivity (user report: warmth/brightness barely visible; presets sometimes appeared to do nothing). | Overhauled in this session — see the Look-panel section of the handover entry for the preset list and per-slider math. |

## Noted, deferred (with reasoning)

- **Toolbar overflow on very narrow windows**: `.bar` is `nowrap`; below
  ~900 px the right tools could clip. Real fix is responsive collapse of
  labeled groups into menus — worth doing with actual window-size testing on
  the XPS half-snap, not blind. (P2)
- **Trash panel cells**: no context menu (restore/delete are header buttons;
  acceptable, but a per-item menu would be nicer). (P3)
- **Cache size / library stats in Settings**: still the standing P0/P3 pair
  from the July audit (GC + visibility). Untouched here — it's a backend
  work item, tracked in BACKLOG.md.
- **Loupe photo zoom**: Focus view fits-to-stage only; no 1:1/zoom toggle.
  Lightroom-parity gap worth a dedicated pass (pinch/scroll zoom, pan,
  Z key). (P2, added to backlog)
- **Icon language**: mixed text-glyph/SVG iconography is consistent enough
  in-app, but a single SVG icon set would be the polished end state. Only
  worth it as part of a deliberate visual refresh with screenshots in the
  loop. (P3)
